Тесты нужно проводить при отключенном файле initial_data.json.

Если фикстуры оставить включенными, то при проведении тестов, Django начнет заполнять ими тестовую БД.

Одним из условий проверочного задания было реализовать с помощью сигналов запись в БД информации 
о создании/редактировании/удалении записей моделей.
Для этого была создана модель, из которой Django автоматически создает  таблицу в БД.

Когда создается тестовая база данных, и она начинает заполнятся данными из фикстур,
модели начинают посылать сигналы о новых записях.
Однако,поскольку процесс формирования еще не завершился, таблица, в которую записывается приходящая информация,
еще не существует-в результате выдается ошибка.
